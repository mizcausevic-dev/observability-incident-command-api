# Security Policy

## Supported Versions

This repository is a portfolio project and does not provide formal version support. Security-minded fixes will be applied on the default branch when relevant.

## Reporting a Vulnerability

If you discover a vulnerability in this project, please avoid posting exploit details in a public issue.

Instead:

1. Contact the repository owner directly through GitHub.
2. Share reproduction steps, affected routes, and any proof-of-concept data.
3. Allow time to validate and remediate before public disclosure.

## Security Posture

This project demonstrates:

- environment-based configuration
- centralized error handling
- request validation with Zod
- CI validation
- dependency update automation
- static code scanning with CodeQL
